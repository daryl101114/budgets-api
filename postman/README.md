# Postman Collection — Budget API

## Structure

```
postman/
  _meta.json                     — Collection info and all collection variables
  01_auth.json                   — Auth endpoints (register, login)
  02_wallets.json                — Wallet CRUD
  03_transactions.json           — Transaction CRUD
  04_transaction_categories.json — Transaction category CRUD
  05_budgets.json                — Budget and allocation management
  06_import.json                 — CSV import batch flow
  07_auth_edge_cases.json        — Auth error scenarios
  build.py                       — Assembles all modules into the importable collection
  README.md                      — This file
```

The file imported into Postman is `Budget API.postman_collection.json` at the project root. **Never edit that file directly** — it is generated by `build.py` and will be overwritten.

---

## Workflow

### Making changes

1. Open the relevant module file (e.g. `06_import.json` for import endpoints)
2. Add, edit, or remove requests inside the `"item"` array
3. Run the build script to regenerate the collection:

```bash
python3 postman/build.py
```

4. Re-import `Budget API.postman_collection.json` into Postman

### Adding a new module

1. Create a new file following the naming convention: `08_my_module.json`
2. Use this skeleton:

```json
{
  "name": "My Module",
  "item": []
}
```

3. Add your requests to the `"item"` array (see any existing module for examples)
4. Run `python3 postman/build.py` — the build script picks up files ordered by their numeric prefix automatically

### Adding a new collection variable

1. Open `_meta.json`
2. Add an entry to the `"variable"` array:

```json
{ "key": "myNewVar", "value": "" }
```

3. Run `python3 postman/build.py`

---

## Running the collection

1. Make sure the API is running: `dotnet run` from the project root
2. In Postman, import `Budget API.postman_collection.json`
3. Run **Auth → Authenticate** first — the test script extracts the JWT from the `Set-Cookie` header and stores it in the `token` variable
4. Subsequent requests use `Authorization: Bearer {{token}}` automatically
5. Test scripts chain IDs between requests (e.g. Create Wallet stores `walletId`, which is used by all wallet sub-requests)

### Recommended run order

```
Auth → Authenticate
Wallets → Create Wallet
Transactions → Create Transaction
Transaction Categories → Get All Categories
Budgets → Create Budget
Import → Upload CSV
... (continue down each folder in order)
Auth Edge Cases
```

## Request body conventions

| Field | Convention |
|-------|-----------|
| Auth header | `Authorization: Bearer {{token}}` on all protected routes |
| IDs in URLs | `{{walletId}}`, `{{transactionId}}`, `{{budgetId}}`, etc. |
| File uploads | `body.mode = "formdata"`, `type = "file"` — select the file manually in Postman |
| Bulk ID arrays | Stored as JSON array strings in variables (e.g. `{{approveIds}}`) and interpolated directly into the raw body |

## Collection variables

| Variable | Set by |
|----------|--------|
| `baseUrl` | Pre-set to `http://localhost:5038` |
| `token` | Auth → Authenticate test script |
| `walletId` | Wallets → Create Wallet test script |
| `transactionId` | Transactions → Create Transaction test script |
| `categoryId` | Transaction Categories → Create Custom Category test script |
| `systemCategoryId` | Transaction Categories → Get All Categories test script |
| `budgetId` | Budgets → Create Budget test script |
| `budgetId2` | Budgets → Create Second Budget test script |
| `allocationId` | Budgets → Add Allocation test script |
| `importBatchId` | Import → Upload CSV test script |
| `importBatchId2` | Import → Upload CSV Again test script |
| `approveIds` | Import → Get Duplicate Batch Detail test script |
| `rejectIds` | Import → Get Duplicate Batch Detail test script |
